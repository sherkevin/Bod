## 1. 产品概述

### 1.1 产品定位

**Bod** 是一款基于 AI 的个人健身助手，通过视觉识别、语音交互和智能算法，为用户提供个性化的健身指导和全程陪伴。核心理念是"方便、快捷、好用"。

### 1.2 产品愿景

让每个人都能拥有属于自己的私人健身教练，随时随地享受专业的健身指导，降低健身门槛，培养健康的生活习惯。

### 1.3 核心价值

| 价值点 | 说明 |
|--------|------|
| **个性化** | 根据用户身体状况、目标和可用器械定制专属方案 |
| **智能化** | AI 教练自动识别器械、生成计划、跟踪进度 |
| **便捷化** | 拍照识别器材，语音交互记录，一键打卡 |
| **激励化** | 每日督促、鼓励反馈、成就系统保持动力 |
| **陪伴式** | AI 教练每日关注进度，像真人教练一样跟进 |

### 1.4 目标用户

| 用户类型 | 占比 | 特征 | 需求 |
|----------|------|------|------|
| **健身新手** | 40% | 缺乏健身知识，不知从何开始 | 需要基础指导、动作教学、入门计划 |
| **有经验者** | 40% | 有一定基础，需要计划和监督 | 需要系统计划、进度跟踪、突破瓶颈 |
| **进阶爱好者** | 20% | 长期训练，追求科学训练 | 需要数据分析、周期化训练、精准调节 |

## 2. 用户角色与场景

### 2.1 核心用户故事

#### 场景一：首次使用 - 器材识别
> **用户小明** 第一次使用 Bod，他刚办了健身卡，面对各种器材不知如何下手。
> 1. 打开应用，拍摄健身房的几张照片
> 2. AI 自动识别出：哑铃、杠铃、深蹲架、跑步机等器材
> 3. 生成"我的器材清单"，小明可以手动补充或修正
> 4. 基于器材清单，AI 为小明推荐合适的训练计划

#### 场景二：制定计划 - 个人目标
> **用户小红** 想要减脂塑形，每周能去健身房 3 次。
> 1. 输入个人信息：身高、体重、年龄、性别
> 2. 选择健身目标：减脂、增肌、塑形、增强体能
> 3. 设置训练频率：每周 3 次，每次 60 分钟
> 4. AI 教练根据她的情况和器材，生成 8 周训练计划
> 5. 小红可以预览和调整计划

#### 场景三：训练执行 - 语音交互
> **用户小李** 正在进行力量训练，手上沾满镁粉。
> 1. 到了训练时间，手机弹出系统通知："该训练了！今天练胸"
> 2. 打开应用，开始今天的训练
> 3. 完成一组卧推后，长按麦克风按钮："做了 12 个，重量 60 公斤，感觉中等"
> 4. AI 教练语音回复："做得好！下一组建议 12-15 个，保持这个重量"
> 5. 也可以点击"语音通话"按钮，进入连续对话模式，像打电话一样交流

#### 场景四：每日跟进 - 进度更新
> **用户小王** 今天状态不好，想调整训练。
> 1. AI 教练每天固定时间发来问候："今天感觉怎么样？训练顺利吗？"
> 2. 小王语音回复："今天有点累，想休息一下"
> 3. AI 教练："理解你，休息也是训练的一部分。明天计划调整一下强度，好吗？"
> 4. 小王也可以随时更新目标、身体状态，AI 会相应调整计划

#### 场景五：打卡督促 - 激励系统
> **用户小张** 连续一周没去健身房了。
> 1. AI 教练发来通知："你已经 3 天没训练了，要不要今天动一动？"
> 2. 小张打开应用，完成今日训练后打卡
> 3. 屏幕显示鼓励动画："连续训练 3 天！保持这个节奏！"
> 4. 积分增加，解锁新成就徽章

## 3. 产品形态

### 3.1 产品形态选择

**PWA 网页应用**（Progressive Web App）

| 考量维度 | PWA | 小程序 | 原生 App |
|----------|-----|--------|----------|
| 开发成本 | 低 | 中 | 高 |
| 跨平台 | ✅ iOS/Android/PC | ❌ 依赖平台 | ❌ 需分别开发 |
| 安装门槛 | 无需安装 | 扫码即用 | 需下载安装 |
| 离线支持 | ✅ | ✅ | ✅ |
| 系统通知 | ✅ | ✅ | ✅ |
| 语音权限 | ✅ | 有限 | ✅ |
| 摄像头权限 | ✅ | ✅ | ✅ |
| 更新发布 | 即时 | 需审核 | 需审核 |

### 3.2 PWA 核心特性

```
┌─────────────────────────────────────────────────────────┐
│                    Bod PWA 应用                          │
├─────────────────────────────────────────────────────────┤
│  添加到主屏幕  →  像原生 App 一样的图标和启动体验        │
│  离线缓存      →  没网也能查看计划和记录训练             │
│  系统通知      →  训练提醒、每日问候不打烊               │
│  后台同步      →  训练数据自动同步，不担心丢失           │
│  更新无声      →  打开即用最新版本，无需手动更新          │
└─────────────────────────────────────────────────────────┘
```

### 3.3 支持平台

| 平台 | 支持状态 | 说明 |
|------|----------|------|
| iOS Safari | ✅ | iOS 11.4+，支持添加到主屏幕 |
| Android Chrome | ✅ | 完整支持，可安装为桌面应用 |
| 桌面浏览器 | ✅ | Chrome/Edge/Firefox/Safari |
| 微信内置浏览器 | ⚠️ | 可用，但无法添加到桌面 |

### 4.2 P0 核心功能（MVP）

#### 4.2.1 用户档案管理

**功能描述**：用户录入和维护个人基本信息及身体数据。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 昵称 | 文本 | 是 | 用于教练称呼用户 |
| 性别 | 选择 | 是 | 男/女 |
| 年龄 | 数字 | 是 | 用于计算基础代谢 |
| 身高 | 数字 | 是 | 单位：cm |
| 体重 | 数字 | 是 | 单位：kg，可更新 |
| 健身水平 | 选择 | 是 | 新手/初级/中级/高级 |
| 健身目标 | 选择 | 是 | 减脂/增肌/塑形/增强体能/保持健康 |
| 训练频率 | 数字 | 是 | 每周训练次数 |
| 每次时长 | 数字 | 是 | 每次训练分钟数 |
| 受伤史 | 文本 | 否 | 影响训练计划的伤病记录 |
| 健身房位置 | 文本 | 否 | 用于判断训练场所 |

**交互流程**：
```
注册/登录 → 引导填写档案 → 选择目标 → 生成初始计划
```

#### 4.2.2 器材识别与管理

**功能描述**：通过拍照识别健身器材，建立用户的器材清单。

**核心流程**：

```
┌──────────────────────────────────────────────────────────┐
│  步骤1：拍照识别                                           │
│  • 用户进入"我的器材"页面                                  │
│  • 点击"拍照添加"按钮                                      │
│  • 拍摄健身房照片（支持连续拍摄多张）                      │
│  • 上传照片到 VLM (视觉语言模型)                           │
│  • AI 识别照片中的健身器材                                 │
├──────────────────────────────────────────────────────────┤
│  步骤2：确认编辑                                           │
│  • 展示识别结果列表                                        │
│  • 用户可确认/删除/补充器材                                │
│  • 可手动添加数量和重量信息（如：哑铃 5-30kg）             │
│  • 保存到器材清单                                          │
├──────────────────────────────────────────────────────────┤
│  步骤3：后续更新                                           │
│  • 随时可以拍照添加新器材                                  │
│  • 手动编辑器材信息                                        │
│  • 标记常用/不常用器材                                     │
└──────────────────────────────────────────────────────────┘
```

**支持识别的器材类别**：

| 类别 | 示例器材 |
|------|----------|
| 自由重量 | 哑铃、杠铃、壶铃、药球 |
| 固定器械 | 胸推机、高位下拉、腿举机、史密斯机 |
| 有氧设备 | 跑步机、椭圆机、动感单车、划船机 |
| 功能训练 | 瑜伽垫、弹力带、TRX、波速球 |
| 辅助器材 | 泡沫轴、引体向上杆、健身球 |

#### 4.2.3 智能训练计划生成

**使用场景**：
- 长按麦克风按钮说话

└──────────────────────────────────────────────────────────┘

**使用场景**：
- 点击"语音通话"进入连续对话模式
- 像打电话一样与 AI 教练实时交流
- AI 主动提问引导对话
- 适合训练前后的深入交流、答疑解惑

**AI 教练回复逻辑**：
```
用户输入 → 语音识别 (ASR) → 意图分析 → 生成回复 → 语音合成 (TTS)

意图分类：
├── 数据记录: 提取 {重量, 次数, RPE} → 保存并反馈建议
├── 请求指导: 动作问题 → 解释要点 + 鼓励
├── 表达感受: 累/痛/不想练 → 情感支持 + 调整建议
├── 咨询问题: 训练相关 → 专业解答
└── 其他闲聊: 正向回应 + 引回训练
```

#### 4.2.5 提醒与打卡系统

**功能描述**：系统在预定时间推送训练提醒，用户完成训练后打卡，记录连续天数。

**提醒设置**：
```
提醒类型:
├── 训练提醒: 计划训练时间前 30 分钟
├── 每日问候: 每天固定时间 (如 20:00)
├── 未训练提醒: 超过计划时间未打卡
└── 连续中断提醒: 超过 3 天未训练
```

**提醒方式**：
- 使用 Web Push API 发送系统通知
- 通知包含：训练主题、快速开始按钮
- 点击通知直接进入今日训练页面

**打卡流程**：
```
┌──────────────────────────────────────────────────────────┐
│  完成今日训练                                              │
│                                                           │
│  ✅ 训练完成                                              │
│                                                           │
│  今日训练: 胸部+三头肌                                    │
│  训练时长: 55 分钟                                        │
│  完成组数: 18 组                                          │
│                                                           │
│  [ 打卡 ]                                                 │
│                                                           │
├──────────────────────────────────────────────────────────┤
│  打卡成功！                                                │
│                                                           │
│  🔥 连续训练 12 天                                         │
│  🏆 解锁成就: "坚持不懈"                                   │
│  💪 这是你连续第二周全勤！                                 │
│                                                           │
│  [ 分享成就 ]              [ 完成 ]                        │
└──────────────────────────────────────────────────────────┘
```

**连续天数规则**：
- 按照计划日期打卡算完成
- 允许补打卡（当天 24 点前）
- 休息日不打卡不影响连续天数
- 连续天数以完成计划周计算

#### 4.2.6 每日跟进与目标更新

**功能描述**：AI 教练每天主动询问训练进度，用户可随时更新身体状态和目标。

**每日问候时机**：
- 默认时间：用户设置的每日问候时间
- 智能调整：根据用户通常训练时间自适应

**问候内容模板**：
```
根据用户情况生成个性化问候：

类型1 - 训练日训练前:
"今天我们要练背部，准备好了吗？上次你硬拉突破了，今天试试能不能再进步！"

类型2 - 训练日训练后:
"今天感觉怎么样？背部训练强度还可以接受吗？记得记录下今天的感觉。"

类型3 - 休息日:
"今天是休息日，好好恢复！如果有时间，可以做些拉伸放松肌肉。"

类型4 - 长时间未训练:
"好久不见你了！是不是遇到什么困难？需要我帮你调整一下计划吗？"
```

**用户可更新的内容**：
| 更新项 | 触发方式 | AI 响应 |
|--------|----------|---------|
| 身体状态 | 主动说明/询问 | 调整训练强度 |
| 体重变化 | 更新档案数据 | 重新评估进度 |
| 目标调整 | 更新健身目标 | 重新生成计划 |
| 受伤情况 | 说明不适部位 | 调整避免受伤动作 |
| 训练时间 | 更新可用时间 | 调整训练安排 |

#### 4.2.7 AI 教练风格系统

**功能描述**：用户可选择不同的教练风格，AI 的语气和鼓励方式会相应调整。

**教练风格选项**：

| 风格 | 定位 | 语言特点 | 适用人群 |
|------|------|----------|----------|
| 严厉型 | 硬核教练 | 直接、严格、不废话 | 需要外部压力的用户 |
| 鼓励型 | 温暖教练 | 温柔、正向、夸奖 | 需要情感支持的用户 |
| 幽默型 | 活泼教练 | 风趣、轻松、开玩笑 | 喜欢轻松氛围的用户 |
| 专业型 | 科学教练 | 客观、数据、解释 | 追求科学训练的用户 |
| 混合型 | 平衡教练 | 根据情况灵活切换 | 大多数用户 |

**风格体现示例**：

场景：用户说"太累了，不想练了"

```
严厉型: "累就是进步的开始！别找借口，动起来！"
鼓励型: "辛苦了，但今天完成了你会更自豪的。我们一起慢慢来好吗？"
幽默型: "累了？想想你办卡时交的钱，那可是真金白银啊！"
专业型: "疲劳是正常反应，可以适当降低强度，但保持训练节奏更重要。"
```

#### Sprint 1: 基础框架 (2周)

| 基础组件开发 | UI 组件库搭建 |
| 用户认证系统 | 注册/登录/JWT |

#### Sprint 2: 核心功能 - 用户与器材 (2周)

| 任务 | 产出 |
|------|------|
| 用户档案管理 | 个人信息录入/编辑 |
| 器材识别接口 | VLM 模型集成 |

| 任务 | 产出 |

| 计划展示页面 | 周/月计划视图 |
| 计划调整功能 | 手动/自动调整 |

#### Sprint 4: 核心功能 - 训练执行 (3周)

| 任务 | 产出 |
|------|------|

| 语音识别集成 | ASR 模型接入 |
| 语音合成集成 | TTS 模型接入 |
| 按住说话交互 | 语音按钮组件 |
| 语音通话模式 | 连续对话界面 |

#### Sprint 5: 核心功能 - 提醒打卡 (2周)

| 任务 | 产出 |

### 8.2 里程碑

| 里程碑 | 时间点 | 验收标准 |
|--------|--------|----------|
| M1: MVP 发布 | 第 13 周 | 完成核心功能闭环，可端到端使用 |

### 8.3 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| AI 模型资源不足 | 高 | 使用云 API 作为备选，Int4 量化降低需求 |
| 语音识别准确率 | 中 | 收集用户反馈数据，持续优化模型 |
| PWA 兼容性问题 | 中 | 提供 Web 降级方案，确保基本可用 |
| 用户获取成本 | 高 | 依靠口碑传播，分享功能设计 |
